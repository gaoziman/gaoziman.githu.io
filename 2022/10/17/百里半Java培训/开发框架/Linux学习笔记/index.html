Linux 目录结构

1.linux的文件系统是采用级层式的树状目录结构，在此结构中的最上层是根目录/，然后在次目录下创建其他目录。

2./bin [常用] (/usr/bin /usr/local/bin) 是Binary的缩写，这个目录存放着最经常使用的命令

3./sbin (/usr/sbin /usr/local/sbin) s就是Super User的意思，这里存放的是系统管理员使用的系统管理程序。

4./home [常用] 存放普通用户的主目录，在Linux中每一个用户都有自己的目录，一般该目录是以用户的账户名命名。

5./root [常用] 该目录为系统管理员，也称作超级权限者的用户主目录

6./lib 系统开机所需要的最基本的动态连接共享库，其作用类似于Windows里面的DLL文件,几乎所有的用户程序都需要用到这些共享库。

7./lost+found 这个目录一般是空的，当系统非法关机后，这里面就存放了一些文件

8./etc [常用] 所有的系统管理所需要的配置文件和子目录，比如安装Mysql数据库 my.conf

9./usr [常用] 用户的很多应用程序和文件都放在这个目录下，类似于windows下的program.file目录

10./boot [常用] 存放的是启动Linux时使用的一些核心文件，包括一些连接文件和镜像文件

11./proc [不能动] 这个目录是一个虚拟的目录，他是系统内存的映射，访问这个目录来获取系统信息

12./srv [不能动] service的缩写，该目录存放一些服务器启动之后需要提取的数据

13./sys [不能动] 这是一个linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs =>[不能动]

14./tmp 这个目录是用来存放一些临时文件的

15./dev 类似于windows的设备管理器，把所有的硬件用文件的形似储存

16.media [常用] linux系统会自动识别一些设备，例如u盘，光驱等等，当识别后，linux会把识别的设备挂载到这个目录下

17./mnt [常用] 系统提供该目录是为了让用户临时挂载别的文件系统，我们可以将外部的存储挂载在/mnt上，然后进入该目录就可以查看里面的内容。

18./opt 这是给主机额外安装软件所存放的目录。

19./usr/local [常用] 这是另一个给主机额外安装软件所安装的目录。一般是通过编译源码方式安装的程序。

20./var [常用] 这个目录中存放着在不断扩充的东西，习惯将经常被修改的目录放在这个目录下。包括各种日志文件。

21./selinux[security-enhanced linux] SElinux 是一种安全子系统，他能控制程序只能访问特定文件，有三种工作模式，可以自行设置。

开机、重启和用户登录注销

  • 关机&重启命令

    • 基本介绍

      1. shutdown -h now 立刻进行关机

      2. shutdown -h 1 1分钟后关机

      3. shutdown -r now 现在重启计算机
      4. halt 立刻关机
      5. reboot 现在重启计算机
      6. sync 把内存的数据同步到磁盘

    • 注意细节

      • 1.不管是重启系统还是关闭系统，首先要运行sync 命令，把内存中的数据写入磁盘中。
      • 2.目前的 shutdown/reboot/halt 等命令均在关机前进行了sync ，但是避免失误，最好在工作中进行手动sync。

  • 用户登录和注销

    • 基本介绍
      • 1.登录时尽量别用root用户登录，因为它是系统管理员，最大的权限，避免失误操作。可以利用普通用户登录，登录后再用su - +用户名 命令来切换成系统管理员身份。
      • 2.在提示符下输入logout 即可注销用户。
    • 使用细节
      • 1.logout 注销指令在图形运行级别无效，在运行级别 3 下有效。

用户管理

  • 一.基本介绍
    • linux系统是一个多用户多任务的操作系统，任何一个要使用系统资源的用户，都必须首先向系统管理员申请一个账号，然后以这个账号的身份进入系统。
  • 二.添加用户
    • 1.基本语法：useradd + 用户名。
    • 2.细节说明：当创建用户成功后，会自动的创建和用户相同的家目录,也可以通过useradd -d + 指定的目录名 + 新的用户名 给新的用户创建指定的家目录。
  • 三.指定修改密码
    • 基本语法：passwd + 用户名
  • 四.删除用户
    • 1.基本语法：userdel + 用户名
    • 2.删除用户及用户主目录userdel -r + 用户名
  • 五.查询用户信息指令
    • 基本语法：id + 用户名
  • 六.切换用户
    • su - 用户名
      • 细节说明：从权限高的用户切换到权限低的用户，不需要用户密码，反之需要,当需要返回到原来用户时，使用exit\logout 指令。
  • 七.查看当前用户/登录用户
    • 基本语法who am i
  • 八.用户组
    • 介绍：系统可以对有共同性/权限的多个用户进行统一管理
    • 新增组：
      • 指令groupadd + 组名
    • 删除组：
      • 指令groupdel + 组名
    • 增加用户时直接给加上组
      • 指令 : useradd -g 用户组 用户名
      • 例如：增加一个用户 zwj ，直接将他指定到 wudang 组
        • 第一步先创建组：groupadd wudang
        • 第二步创建用户并将用户放置到改组：useradd -g wudang zwj
    • 修改用户的组 :
      • 指令 ：usermod -g 用户组 用户名
      • 案例演示：
        • 第一步创建一个组：mojiao groupadd mojiap
        • 第二步把 zwj 放入到 mojiao ：usermod -g mojiao zwj
  • 九.用户和组相关文件
    • /etc/passwd 文件
      • 用户 user 的配置文件，记录用户的各种信息
      • 每行的含义：用户名：口令：用户标识号：组标识号：注释性描述：主目录：登录Shell
    • /etc/shadow 文件
      • 口令的配置文件
      • 每行的含义：登录名：加密口令：最后一次修改时间：最小时间间隔：最大时间间隔：警告时间：不活动时间：失效时间：标志
    • /etc/group 文件
      • 组 (group) 的配置文件，记录 Linux 包含的组的信息
      • 每行的含义：组名：口令：组标识号：组内用户列表

实用指令

一.指定Linux运行级别

  • 基本介绍：

    • 运行级别说明：

      0 ：关机

      1 ：单用户[找回丢失密码]

      2 : 多用户状态没有网络

      3 ： 多用户状态有网络服务

      4 : 系统未使用保留给用户

      5 ： 图形界面

      6 ： 系统重启

    • 常用运行级别是 3 和 5 ，也可以默认运行级别。

  • 应用实例：

    • 命令：init [0123456] : 通过 init 来切换不同的运行级别，比如 init 3 ,就是传统的dos界面。
    • 第二种方式：systemctl set-default TARGET.target
二.帮助指令
  • 1. man 获得帮助信息
    • 基本语法：man[命令或者配置文件] (功能描述：获取帮助信息)
    • 案例：查看ls 命令的帮助信息，man ls
    • 在Linux下，隐藏文件是以 . 开头，选项可以组合使用，比如 ls -al ,比如 ls -al/root
  • 2. help 指令
    • 基本语法 ：help + 命令 ，(功能描述：获取shell内置命令的帮助信息)
    • 应用实例：查看 cd 命令的帮助信息 ：help cd
三.文件目录类

  • 1. pwd 命令

    • 基本语法：pwd (功能描述 ： 显示当前工作目录的绝对路径)
    • 应用实例：案例：显示当前工作目录的绝对路径

  • 2. ls 指令

    • 基本语法：ls [选项] [目录或者文件]
    • 常用选项：
      • -a : 显示当前目录所有的文件和目录，包括隐藏的
      • -l : 以列的方式显示信息

  • 3. cd 指令

    • 基本语法：cd [参数] (功能描述：切换到指定的目录)

    • 理解：绝对路径和相对路径

    • cd ~ 或者 cd : 回到自己的家目录，比如 你是’root’ , cd~ 到 /root

    • cd .. 回到当前目录的上一级目录

    • 应用实例：

      • 案例1：使用绝对路径切换到 root 目录， cd /root
      • 案例2：使用相对路径到 root 目录，比如在 /home/tem, cd ../../root
      • 案例3：表示回到当前目录的上一级，cd ..
      • 案例4：回到家目录，cd ~

    • mkdir 指令用于创建目录：
      • 基本语法：mkdir [选项] 要创建的目录名称
    • 常用选项
      • -p : 创建多级目录
    • 应用实例：
      • 案例1：创建一个目录 /home/dong
        • 指令：mkdir /home/dong
      • 案例2：创建多级目录 /home/animal/taiger
        • 指令 mkdir -p /home/animal/taiger

  • 5. rmdir 指令

    • 基本语法：rmdir [选项] 要删除的目录
    • 应用实例：
      • 案例：删除一个目录 /home/dog
        • 指令：rmdir /home/dog
    • 使用细节：
      • rmdir 删除的是空目录，如果目录下有内容时无法删除
      • 提示：如果需要删除非空目录，需要使用rm -rf 要删除的目录
      • 比如：rm -rf /home/animal

  • 6. touch 指令

    • touch 指令创建空文件
    • 基本语法：touch 文件名称
    • 应用案例：在/home 目录下，创建一个空文件 hello.txt
      • 指令 touch hello.txt

  • 7.cp 指令

    • cp 指令拷贝文件到指定目录
    • 基本语法：cp [选项] source dest
    • 常用选项：
      • -r :递归复制整个文件
    • 应用实例：
      • 案例1：将 /home/hello.txt 拷贝到 /home/bbb 目录下
        • 指令：cp /home/hello.txt /home/bbb
      • 案例2：递归复制整个文件夹，举例，比如将 /home/bbb 整个目录，拷贝到 /opt
        • 指令：cp -r /home/bbb /opt
    • 使用细节：如果目标目录下已存在该文件名，那么该操作会覆盖该文件
      • 强制覆盖不提示的方法：\cp 在cp 前加上\
      • 比如：\cp -r /home/bbb /opt

  • 8. rm 指令

    • 说明：rm 指令移除文件或者目录
    • 基本语法：rm [选项] 要删除的文件或者目录
    • 常用选项：
      • -r ：递归删除整个文件夹
      • -f ：强制删除，不在提示
    • 应用实例：
      • 案例1：将/home/hello.txt 删除，rm /home/hello.txt
      • 案例2：递归删除整个文件夹 /home/bbb, rm -rf /home/bbb
    • 使用细节：强制删除不提示的方法：带上 -f 参数即可

    • mv 移动文件与目录或者重命名

      • mv oldNameFile newNameFile (功能描述：重命名)
      • mv /temp/movefile /targetFolder (功能描述：移动文件)

    • 基本语法：
      • ls -l > 文件 (功能描述：显示的列表内容写入文件中 （覆盖写）)
      • ls -al >> 文件 (功能描述：列表显示的内容追加到文件中末尾)
      • cat 文件1 > 文件2 (功能描述：将文件1中的内容 覆盖 文件2的内容)

      • 在 /home 目录下创建一个软链接 myroot ，连接到 /root 目录
      • 指令：ln -s /root /home/myroot
    • 删除软链接：rm -f /home/myroot
    • 细节说明：当我们使用 pwd 查看目录时，仍然看到的是软链接所在目录

    • 查看已经执行过的历史命令，也可以执行历史命令
    • 基本语法：history (功能描述：查看已经执行过的历史命令)
    • 实用案例：

四.时间日期类
  • 1. date 指令-显示当前日期

四.权限的基本介绍

  • 基本介绍：

    • ls -l 或者 ll 显示所在目录的所有文件及目录的权限 -rwxrw-r-- 1 root root 1213 Feb 2 09:39 abc

    • 0~9位说明

    • 第 0 位说明文件类型(d, -, l, c, b)

      • l 是链接，Linux中的软链接类似于Windows中的快捷键
      • d 是目录，相当于windows中的文件夹
      • c 是字符设备文件，鼠标，键盘
      • b 是块设备，比如硬盘
      • - 是普通文件，如 .txt

    • 第 1~3 位确定 所有者 (该文件的所有者) 拥有改文件的权限 —-User

    • 第 4~6 位确定 所属组 (同用户组的) 拥有改文件的权限—-Group

    • 第 7~9 位确定 其它用户 拥有该文件的权限—-Other

    • 可以使用数字表示为：r = 4, w = 2, x = 1,所有 rwx = 4 + 2 + 1 = 7,数字可以进行组合

    • 其他说明

      | ———————- | —————————————————————————————————- |
      | ———————- | —————————————————————————————— |
      | 1 | 文件：硬链接数或目录：子目录数 |
      | root | 用户 |
      | root | 组 |
      | 1213 | 文件大小(字节），如果是文件夹，显示 4096 字节 |
      | Feb 2 09:39 | 最后修改的时间 |
      | abc | 文件名 |

  • rwx 权限详解

    • rwx 作用到文件
      • [r] ：代表可以读 read : 可以读取，查看
      • [w] : 代表可写 write : 可以修改，但是不代表可以删除该文件，删除一个文件的前提条件是对该文件所在目录有写的权限，才能删除该文件
      • x : 代表可执行 execute : 可以被执行
    • rwx 作用到目录
      • [r] ：代表可以读 read : 可以读取，ls 查看目录内容
      • [w] : 代表可以写 write : 可以修改，对目录内 创建+删除+目录重命名
      • x : 代表可执行 execute : 可以进入该目录

      2. Linux 采用了一种叫 ”挂载“ 的处理方法，它的整个文件系统中包含了一整套的文件和目录，且将一个分区和一个目录联系起来。这时要载入的一个分区将使它的储存空间在一个目录下获得。

一.显示系统执行的进程

      • STIME ：进程启动时间
      • TTY ：完整的终端名称
      • TIEM ：CPU 时间
      • CMD ：启动进程所用的命令和参数

      • rpm -qf /etc/passwd

      • 如果其他软件包依赖于你要卸载的软件包，卸载时则会产生错误信息
        • 指令：rpm -e foo
        • removing these packages would break dependencies:foo is needed by bar-1.0-1
      • 如果我们就是要删除foo这个rpm包，可以增加参数 —nodeps ,就可以强制删除，但是一.般不推荐这样做，因为依赖于该软件包的程序可能无法运行
        • 指令：rpm -e --nodeps foo

mysql5.7的安装

  • 安装步骤：

      1. mkdir /opt/mysql : 新建文件夹/opt/mysql，并cd进去

      2. 通过 xftp6 将mysql传至新目录—->上传到 /opt/mysql 下

      3. cd /opt/mysql：去已经有 mysql 的安装包下

      4. 运行 tar -xvf mysql-5.7.26-1.el7.x86_64.rpm-bundle.tar ：对安装包进行解压

      5. centos7.6自带的类mysql数据库是mariadb，会跟mysql冲突，要先删除。

        1. 运行 rpm -qa|grep mari，查询mariadb相关安装包
        2. 运行 rpm -e --nodeps mariadb-libs，卸载
        3. 运行 rpm -e --nodeps marlisa，卸载

      6. 然后开始真正安装mysql，依次运行以下几条

        1. rpm -ivh mysql-community-common-5.7.26-1.el7.x86_64.rpm
        2. rpm -ivh mysql-community-libs-5.7.26-1.el7.x86_64.rpm
        3. rpm -ivh mysql-community-client-5.7.26-1.el7.x86_64.rpm
        4. rpm -ivh mysql-community-server-5.7.26-1.el7.x86_64.rpm

      7. 运行 systemctl start mysqld.service，启动mysql

      8. 然后开始设置root用户密码

        1. Mysql自动给root用户设置随机密码，运行 grep "password" /var/log/mysqld.log 可看到当前密码

        2. 运行 mysql -u root -p，用root用户登录，提示输入密码可用上述的，可以成功登陆进入mysql命令行

        3. 设置root密码，对于个人开发环境，如果要设比较简单的密码（生产环境服务器要设复杂密码），可以运行

          set global validate_password_policy=0 提示密码设置策略;（validate_password_policy默认值1，）

        4. set password for 'root'@'localhost' =password('hspedu100'); ：hspedu100 为新密码

        5. 运行 flush privileges; 使密码设置生效 （注意里面有 ; ）
